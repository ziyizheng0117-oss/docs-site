这里不再只是放几篇零散文章，而是逐步整理成一份可持续维护的 **Kubernetes 知识地图**。

如果你希望把 K8s 学成一个能推理、能排障、能做架构判断的系统，建议先按下面顺序读。

## 这套专题的学习主线

### 1. 先建立系统观

先理解：

- Kubernetes 本质是声明式控制系统
- Control Plane 与 Worker Node 的职责边界
- API Server、etcd、Scheduler、Controller Manager、kubelet 如何协作

### 2. 再掌握工作负载抽象

重点吃透：

- Pod 为什么是最小运行单元
- Deployment、ReplicaSet、StatefulSet 的差异
- Job、CronJob、DaemonSet 分别适用于什么场景

### 3. 再看流量与服务发现

重点包括：

- Service 的抽象作用
- Ingress / Gateway 的入口治理
- CoreDNS、Headless Service、NetworkPolicy 的边界

### 4. 再看资源、存储与调度

重点包括：

- requests / limits
- QoS、Quota、LimitRange
- PV / PVC / StorageClass
- 调度约束、亲和性、污点与容忍

### 5. 最后补安全与排障

重点包括：

- RBAC / ServiceAccount / Secret
- Pod Security / Admission
- Pending / CrashLoopBackOff / NotReady 的排查方法
- 网络、DNS、存储与权限问题的定位思路
